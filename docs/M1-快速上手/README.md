# M1 工具上手 — 学习路径

> **M1 目标**：从零开始，让芯片工程师能独立使用 Claude Code 完成日常开发任务。

## 推荐学习顺序

| 顺序 | 页面 | 预计耗时 |
|------|------|---------|
| 1 | [1-0-什么是Claude-Code](./1-0-什么是Claude-Code.md) | 阅读 5min |
| 2 | [1-1-安装与启动](./1-1-安装与启动.md) | 阅读+实操 15min |
| 3 | [1-2-基础对话](./1-2-基础对话.md) | 阅读+实操 15min |
| 4 | [1-3-常用命令速查](./1-3-常用命令速查.md) | 阅读 5min |
| 5 | [1-4-权限与沙箱](./1-4-权限与沙箱.md) | 阅读+配置 5min |
| 6 | [1-5-IDE集成](./1-5-IDE集成.md) | 阅读+安装 10min |
| 7 | [1-6-用Claude-Code解决真实问题](./1-6-用Claude-Code解决真实问题.md) | 阅读+实操 15min |
| 8 | [1-7-AI能做什么不能做什么](./1-7-AI能做什么不能做什么.md) | 阅读 5min（附录） |

**总耗时**：约 75 分钟（阅读+实操）。

## 不能跳过的前置页面

- 1-1 不能跳过——没装好就没法继续
- 1-2 不能跳过——不会对话就等于不会用
- 1-6 不能跳过——这是 M1 的实战篇，也是理解 M2 的前提

## 进入 M2 之前

> ⚠️ **确保你已完成 1-6 的课后练习**：用 Claude Code 解决一个你自己项目的真实 bug，跑通"定位 → 修复 → 验证"的完整链路。
>
> M2 会系统讲解 Claude Code 的技能体系。如果你还没有亲身体验过完整工作流，M2 的全景图会缺少"锚点"。
