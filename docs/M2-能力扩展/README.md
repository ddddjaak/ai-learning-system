# M2 能力扩展 — 学习路径

## 概述

M2 回答一个核心问题：**"Claude Code 已经会用了，怎么把价值拉到最大？"**

答案分四层：

| 层级 | 你要解决的问题 | 关键页面 |
|------|--------------|---------|
| **认知层**（知道有什么） | "42 个 skill 各自能做什么？我该关注哪些？" | 2-0, 2-1 |
| **适配层**（让 AI 懂你） | "AI 总编造不存在的 API？怎么让 AI 理解我的项目？" | 2-3, 2-4, 2-5 |
| **自动化层**（让 AI 自己跑） | "能不能提交代码时自动审查？能不能批量跑任务？" | 2-6, 2-7, 2-8 |
| **内置能力层**（Claude Code 原生功能） | "Plan Mode 怎么用？怎么编排多个 Agent？跨会话记忆？" | 2-9, 2-10, 2-11, 2-12 |

> **类比**：M1 教会你开车。M2 教你改装车——加导航（skill 全景图）、调引擎（CLAUDE.md）、装行车记录仪（hooks）、启用自动驾驶（Plan Mode + Workflow + Memory）。

```

2-3 CLAUDE.md 怎么写 ──→ 2-4 怎么创建自己的 skill ──→ 2-5 MCP 插件使用

                               ┌──────────────────────────────┘
                               ↓
               2-6 Hooks 自动化 ──→ 2-7 并行代理 ──→ 2-8 编程方式运行
                                                              │
                               ┌──────────────────────────────┘
                               ↓
    2-9 Plan Mode ──→ 2-10 Workflow 多Agent编排 ──→ 2-11 Memory 记忆系统
                                                              │
                               ┌──────────────────────────────┘
                               ↓
                        2-12 Web Search & Fetch
                                                              │
                               ┌──────────────────────────────┘
                               ↓
    2-13 权限管理体系 ──→ 2-14 斜杠命令速查
```

> **最少必读**：2-0 → 2-1 → 2-3。全景图让你知道有什么，日常融入让你会用，CLAUDE.md 是所有自定义能力的基础。

> **进阶必读**：2-9 → 2-10 → 2-13。Plan Mode 和 Workflow 是最强大的内置能力；权限管理是公司环境必备。
>
> **按需跳读**：对 MCP 感兴趣可以先读 2-5。遇到审查效率瓶颈跳 2-7 和 2-10。总是重讲项目背景跳 2-11。想知道所有斜杠命令跳 2-14。

## 什么时候看哪页（按需速查）

**Skill 相关：**
- 不知道有什么 skill → **2-0**（全景图）
- 知道有 skill 但不知道怎么触发 → **2-1**（ae-skills 融入日常）
- 想封装公司的检查规范为可复用流程 → **2-4**（创建自己的 skill）

**让 AI 更懂你的项目：**
- AI 频繁编造不存在的 API / 文件路径 → **2-3**（CLAUDE.md）
- 每次新会话都要重新解释项目背景 → **2-11**（Memory）
- 想看调用链 / 查官方库文档 → **2-5**（MCP 插件）

**让 AI 自己跑：**
- 想让 AI 在 git commit 时自动做某事 → **2-6**（Hooks）
- 代码量大了，一个个审查太慢 → **2-7**（并行代理）→ **2-10**（Workflow）
- 想接入 CI / 批量自动化 → **2-8**（编程方式运行）

**Claude Code 内置能力：**
- 复杂任务写到一半发现方向错了 → **2-9**（Plan Mode）
- 想让 AI 自己循环直到达到某个标准 → **2-9**（Goal Mode）
- 需要并行审查+验证+批量处理 → **2-10**（Workflow）
- 想查芯片 datasheet / errata / 选型对比 → **2-12**（Web Search）
- 每次弹确认框太烦 → **2-13**（权限管理）
- 想知道有哪些斜杠命令 / 某个命令怎么用 → **2-14**（斜杠命令速查）

## 自学路径建议

### 🚀 最小必读（1 小时）

1. 2-0（全景图）— 10 分钟——翻一遍，知道 42 个 skill 的存在
2. 2-1（ae-skills 融入日常）— 15 分钟——理解什么场景触发什么
3. 2-3（CLAUDE.md 怎么写）— 20 分钟——用你的真实项目写一份 CLAUDE.md
4. 2-9（Plan Mode）— 8 分钟——学会"先规划再执行"

### 📚 完整通道（3 小时）

1. 按推荐阅读顺序从上到下（2-0 → ... → 2-14）
2. 每个含实操的页面在你自己项目上动手做一遍
3. 2-2（se-skills）如果不是 SE 可只看概述，需要时再回来
4. 2-14（斜杠命令速查）不需要从头读——收藏为参考页，遇到不知道的命令回来查

### 🎯 场景驱动（按当前痛点切入）

- **刚接手新项目，代码完全不熟** → 2-5（codegraph 探索代码结构）→ 2-3（写 CLAUDE.md）→ 2-11（Memory 记录发现）
- **每次新会话都要重新解释项目** → 2-11（Memory）→ 2-3（CLAUDE.md 补充静态规则）
- **团队要统一 AI 辅助开发规范** → 2-3（CLAUDE.md）→ 2-4（封装 skill）→ 2-6（Hooks 强制检查）
- **审查效率低，一次只能看一个文件** → 2-7（并行代理）→ 2-10（Workflow 编排）
- **复杂需求老是写到一半跑偏** → 2-9（Plan Mode）
- **想让 AI 自动修到某个标准**（测试全绿 / 警告清零）→ 2-9（Goal Mode）
- **查 datasheet 来回切窗口太烦** → 2-12（Web Search）
- **AI 弹确认框太频繁，想配权限** → 2-13（权限管理）
- **不知道有哪些斜杠命令可用** → 2-14（斜杠命令速查）
- **要交付客户方案，需要 SE 能力** → 2-2（se-skills 融入日常）
