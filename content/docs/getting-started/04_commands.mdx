# 常用命令速查

## 概述

Claude Code 不只是聊天——它有一套 slash 命令系统，用来控制对话状态、切换模型、管理权限。这些命令你每天都会用到。

> **类比**: 如果你把 Claude Code 当编译器，slash 命令就是编译器选项（`-O2`、`-g`、`-Wall`）。你不会每次都用所有的选项，但关键时刻用对一个，效率差 10 倍。

---

## 命令速查表

| 命令 | 作用 | 当你……时用 |
|------|------|-----------|
| `/help` | 列出所有可用命令 | 忘了命令名字 |
| `/clear` | 清空对话历史，重新开始 | 上下文太乱，想换一个新话题 |
| `/compact` | 压缩对话历史，释放空间 | 对话很长，AI 开始"忘事"或响应变慢 |
| `/fast` | 切换为快速模式（更低延迟） | 简单任务不需要深度推理，想快一点 |
| `/model` | 查看/切换模型 | 需要更强的推理能力 or 想省钱用轻量模型 |
| `/config` | 查看/修改 Claude Code 配置 | 改主题、改语言、查看当前配置 |
| `/add-dir` | 添加额外的工作目录 | 项目代码在新目录，需要 AI 能读到 |
| `/init` | 为当前项目生成 CLAUDE.md | 第一次在项目里用 Claude Code，建立项目约束 |
| `/permissions` | 管理权限规则 | 弹窗太多，想批量允许/拒绝某些操作 |
| `/cost` | 查看当前会话的 token 用量和费用 | 想知道今天用了多少钱 |

## 常用命令详解

### `/clear` — 重开一张白纸

**什么时候用**：话题完全切换。比如你刚才在调试 I2C 驱动，现在想讨论 SPI 的架构设计——两个话题无关。清空后 AI 不再保留之前的对话上下文。

```bash
> /clear
# 对话从头开始，所有之前的讨论消失
```

> 对比 `/compact`：`/compact` 会保留关键决策和修改，只是压缩冗余内容。话题切换用途不同——`/clear` 是换话题，`/compact` 是继续同一个话题但省空间。

### `/compact` — 长对话的救命命令

**什么时候用**：对话太长了（比如你 debug 了 30 分钟），AI 开始出现"忘事"迹象——你明明告诉过它某个函数名，它又问你一遍。这时候上下文窗口快满了。

```bash
> /compact
# AI 自动压缩对话历史，保留关键上下文
# 压缩后问一句 "刚才我们讨论到什么程度了？" 确认 AI 还记得
```

**嵌入式场景**：你在调一个 I2C + DMA 的联合 bug，对话里已经涉及了 i2c_driver.c、dma_config.c、中断向量表等 8 个文件。输入 `/compact`，AI 会压缩分析过程但保留每个文件的修改决策。

### `/fast` — 快速模式切换

**什么时候用**：简单操作不需要深度推理。比如"帮我把这 5 个文件的 tab 换成 4 空格"——纯体力活，不需要动脑。

```bash
> /fast
# 切换为快速模式（自动使用更轻量的模型）
# 再输入一次 /fast 切回来
```

### `/model` — 手动选择模型

**什么时候用**：公司供应商提供了多个模型可选。主力模型推理质量高但慢，轻量模型快但简单任务用足够。

```bash
> /model
# 显示当前模型和可用的其他模型
# 按提示选择切换
```

> ⚠️ 公司通过 CC Switch 管理的模型就是你的选项池。如果你只配置了一个模型，`/model` 不会有第二个选择。

### `/add-dir` — 跨目录工作

**什么时候用**：你的项目代码分散在多个目录——比如驱动在 `~/projects/chip/drivers`，测试在 `~/projects/chip/tests`，脚本在 `~/tools/scripts`。AI 需要同时看到所有这些目录。

```bash
> /add-dir ~/tools/scripts
# AI 现在也能读取 scripts 目录下的内容了
```

### `/init` — 为项目建立"说明书"

**什么时候用**：第一次在项目里用 Claude Code。它会分析你的项目结构，生成一个 `CLAUDE.md` 文件——等于给 AI 写了一份"项目说明书"。之后每次 AI 进入这个项目，都会先读这份说明书，理解你的项目约束。

```bash
> /init
# AI 分析项目结构，生成 CLAUDE.md 草稿
# 你确认和修改后保存
```

> 详细用法见 [2-3-CLAUDE.md怎么写.md](#)

## 嵌入式工程师常用命令组合

| 场景 | 命令组合 | 效果 |
|------|---------|------|
| 开始调试 | `/clear` → `cd 到项目` → 描述问题 | 干净上下文，只聚焦当前 bug |
| 长 debug 到一半 | `/compact` → 继续问 | 释放空间但不丢上下文 |
| 审查代码 | `/model` 切主力模型 → `请审查 xxx.c` | 审查用最强模型，确保不遗漏 |
| 批量改代码 | `/fast` → `把 xx 目录的所有 HAL_Delay 改成 vTaskDelay` | 简单替换用快速模式省时间 |
| 跨项目分析 | `/add-dir ../common-lib` → `搜索两个项目的调用关系` | 打破单目录限制 |

## 常见问题

**Q: `/compact` 后会丢信息吗？**

AI 会保留关键上下文（文件路径、修改记录、你的约束条件），但会丢弃对话中冗余的分析过程。压缩后建议先问一句"我们现在讨论到哪一步了？"确认 AI 还记得核心信息。

**Q: `/clear` 和 `/compact` 到底什么时候用哪个？**

换话题 → `/clear`。同一话题继续 → `/compact`。简单的判断标准：如果你说"不聊这个了，换个话题"就该 `/clear`；如果你说"我们刚才在调 I2C，继续"就该 `/compact`。

**Q: 命令可以组合使用吗？**

可以。比如 `/fast` 切换快速模式后继续聊天，`/compact` 后再 `/add-dir` 加目录——互不影响。
