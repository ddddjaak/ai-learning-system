# 什么是 Claude Code

## 概述

**Claude Code 是一个运行在终端里的 AI 编程助手。** 它不是你浏览器里聊天的 ChatGPT，也不是在你敲代码时弹出补全提示的 GitHub Copilot。它直接进入你的项目目录，读取你的代码，理解你的上下文，然后帮你读代码、改代码、跑命令、修 bug——全程在你的终端里完成。

> **类比**: 如果你用过的 IDE（Keil / IAR / VS Code）是"一个有图形界面的编辑器"，那 Claude Code 就是"终端里多了一个能读你全部代码、理解芯片项目上下文、帮你动手改代码的 AI 同事"。你说需求，它动手。

**对芯片工程师来说，Claude Code 直接解决这几个问题：**

| 你平时遇到的问题 | Claude Code 怎么帮 |
|-----------------|-------------------|
| 接手一个遗留驱动代码，看不懂结构 | 让它读代码，用自然语言解释逻辑，画调用链 |
| 调试时 log 看不出问题，怀疑是某段代码 | 让它分析代码找潜在 bug（竞态、溢出、未初始化） |
| review 代码耗时间，审查清单总有遗漏 | 让它做五维度审查（正确性/可读性/架构/安全/性能） |
| 写完功能再写测试，测试写不全 | 让它先写测试再写实现（TDD），确保覆盖边界 |
| 需求模糊就开始写代码，写到一半发现方向错了 | 让它先和你对话澄清需求，再出方案 |

## 一个真实的使用场景

假设你接手了一个遗留的 I2C 驱动，想搞清楚它在做什么、有没有 bug：

```
$ cd ~/projects/motor_ctrl_fw
$ claude                          ← 启动 Claude Code

> 请解释一下 drivers/i2c/i2c_master.c 里的 I2C_Write 函数在做什么

AI 逐行解释函数逻辑、参数含义、错误处理路径

> 这个函数在连续写入时有没有可能导致总线被锁住？

AI 分析出：如果从机 NACK，当前代码没有超时退出机制，可能死等

> 帮我加上超时机制，1ms 轮询，最多等 100ms

AI 直接修改代码（你确认后生效）

> 帮我在 tests/ 下写一个 mock 从机测试，验证超时逻辑

AI 生成测试用例

> 运行刚才的测试

测试通过。
```

从读代码到找到 bug 到修复到验证——全程没离开终端。AI 不是帮你查文档，是直接动手改你的代码。

## 和你知道的工具有什么不同

|  | Claude Code | ChatGPT 网页版 | GitHub Copilot |
|---|---|---|---|
| 运行位置 | 你的终端里 | 浏览器里 | IDE 插件 |
| 能看到你的项目吗？ | ✅ 读整个项目代码 | ❌ 只能看到你复制粘贴的片段 | ⚠️ 当前文件和少量上下文 |
| 能改你的文件吗？ | ✅ 直接编辑（你确认后） | ❌ 只能给建议 | ❌ 只能行内补全 |
| 能跑命令吗？ | ✅ git/build/test 随便跑 | ❌ 不能 | ❌ 不能 |
| 能理解芯片代码吗？ | ✅ C/H/寄存器/驱动都懂 | ✅ 能聊但看不到你的项目 | ⚠️ 只能补全当前行 |
| 用在哪一步最合适？ | 全流程（读→改→测→查→审） | 问概念/查方案/学新知识 | 写重复代码时补全 |

**简单说**：ChatGPT 是"聊天搭子"，Copilot 是"行内补全器"，Claude Code 是"在你终端里帮你干活的 AI 工程师"。

## 你需要知道的关键概念

### 它不是"聊天机器人"，是"代理"（Agent）

Claude Code 的工作方式不是一问一答。它是**代理循环**：

```
你给一个指令
  → AI 自主决定：读哪个文件？搜什么内容？跑什么命令？
  → AI 执行操作（读文件/搜索/编辑/运行）
  → AI 看到结果，决定下一步（继续分析 or 返回给你）
  → 回到第一步，循环直到任务完成
```

所以你不需要一步步告诉它"先读这个文件，再读那个文件"。你可以直接说"帮我把 I2C 驱动的错误处理全部加上超时机制"，它会自己找到所有相关文件，逐个分析，做出修改。

### 它有"工具"

Claude Code 不只是聊天——它能调用工具：

| 工具 | 能力 | 嵌入式场景 |
|------|------|-----------|
| **Read** | 读取你的项目文件 | 读驱动代码、数据手册相关代码、makefile |
| **Write / Edit** | 创建/修改文件 | 修复 bug、添加功能、写测试 |
| **Bash** | 运行终端命令 | 编译、烧录、跑测试、git 操作 |
| **Grep** | 搜索代码内容 | 找所有调用 `HAL_I2C_Master_Transmit` 的地方 |
| **Glob** | 按文件名模式匹配 | 找所有 `*_test.c` 文件 |

### 你要确认它才能动

AI 不会自己偷偷改你的代码。**每次编辑文件、执行可能有副作用的命令之前，Claude Code 都会弹出确认框。** 你可以选择允许、拒绝、或配置为"信任这类操作以后都不用再确认"。这是芯片工程师的安全底线——你始终是决策者，AI 是执行者。

## 常见问题

**Q: 和 ChatGPT 有什么区别？我直接复制代码到 ChatGPT 不是一样吗？**

不一样。ChatGPT 只能看到你复制粘贴的几行代码，看不到项目全貌，不知道你的 HAL 封装、中断配置、寄存器映射。Claude Code 能读整个项目，理解调用关系——分析一个函数时，它会去看被调用的函数、相关的头文件、寄存器定义。

**Q: 我需要会编程才能用吗？会不会被 AI 取代？**

你需要会编程才能判断 AI 的输出是否正确。Claude Code 不会取代你——它让你从"一行一行写"变成"审查 AI 写的对不对"。你的价值从"打字"转移到"做判断"。

**Q: 它能读懂 C 代码和寄存器操作吗？**

能。Claude Code 的训练数据包含大量嵌入式代码，理解 C 语言、HAL、CMSIS、寄存器映射、中断向量、链接脚本。但它不一定知道特定的芯片型号（比如 STM32F407 的某个冷门寄存器地址）——那些你还是得查数据手册。

**Q: 我的代码会被上传到哪里？**

代码会被发送到 Claude API 服务器进行处理，不会用于模型训练。如果你用 CC Switch 配置了公司的 API 供应商，代码会经过那个供应商的服务器。具体数据政策以公司 IT 确认的为准。

**Q: 它免费吗？**

Claude Code CLI 本身免费开源，但调用 AI 模型需要 API Key——公司会提供。你可以通过 CC Switch 切换不同的模型供应商（详见下一篇）。
