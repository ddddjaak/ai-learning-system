    斜杠命令速查¶

    核心价值¶

    Claude Code 有 55+ 个内置斜杠命令（/command）。它们是最快的交互方式——比自然语言描述更快、更精确、更不容易被 AI 误解。

    一句话：斜杠命令是 Claude Code 的快捷键。本页列出所有命令，标注哪些对 AE 工程师最重要。

    使用说明¶

    每个命令标注了三样东西：

    标记 含义
    🔴 每天在用——必学
    🟡 每周在用——推荐学
    🟢 按需使用——知道存在即可
    ⚪ 特定角色/平台——可能用不到

    一、会话管理（8 个）¶

    管理对话的生命周期——开始、压缩、回退、导出。

    命令 用途 AE 场景 频率
    /clear 清空对话历史，开始新会话 排查完一个 bug，开始排查下一个——清掉无关上下文 🔴
    /compact 压缩对话历史，释放上下文空间 长对话排查（1 小时+）→ 上下文快满了 → 压缩后继续。可以加指令："compact 保留 I2C 问题的排查结论" 🔴
    /rewind 回退代码和对话到之前的检查点 AI 改错了 3 个文件 → /rewind 一键回退 → 重新描述需求 🟡
    /rename 重命名当前会话 给会话起有意义的名字："调试-I2C-超时-SCL拉低" 方便以后找回 🟡
    /resume 恢复之前的会话 昨天的调试今天继续——不用重新描述问题 🟡
    /export 导出对话为文本文件 把排查过程导出给同事看、写 bug report 🟢
    /background 将会话放到后台运行 让 AI 在后台跑测试、编译——你继续做别的事 🟢
    /branch 从当前会话分叉 方案 A 和方案 B 都试试——分叉两个会话分别探索 🟢

    二、配置与诊断（8 个）¶

    管理和排查 Claude Code 本身。

    命令 用途 AE 场景 频率
    /permissions 管理 allow/deny/ask 权限规则 配好了就不用每次点确认——详见 2-13 权限管理 🔴
    /doctor 诊断 Claude Code 安装和配置问题 "AI 说 codegraph 不可用" → /doctor 看哪里断了 🟡
    /init 初始化项目的 CLAUDE.md 新项目第一件事：/init → AI 扫描项目 → 生成 CLAUDE.md 🟡
    /config 查看/修改设置 改 theme、改 model、改权限——不用手动编辑 JSON 🟡
    /memory 管理 CLAUDE.md 记忆文件 编辑项目规则、查看自动记忆——详见 2-11 Memory 🟡
    /status 查看账号、模型、系统状态 "我现在用的是什么模型？token 还剩多少？" 🟢
    /model 切换 AI 模型（Opus/Sonnet/Haiku） 简单任务切 Haiku（快/便宜），复杂审查切 Opus（强/贵） 🟢
    /effort 设置推理深度（low~max） 简单格式化 → low。硬件 bug 根因分析 → max 🟢

    三、代码审查与质量（5 个）¶

    内置的代码审查工具——和 ae-skills 的 review 互补。

    命令 用途 AE 场景 频率
    /review 审查 GitHub PR 同事提了个 PR → /review 自动审查 → 输出发现 🟡
    /code-review 审查当前 diff（未提交的改动） 代码写完了，提交前自查 → /code-review low 快速扫一遍 🟡
    /security-review 审查当前 diff 的安全问题 固件改动了 bootloader 或 OTA 逻辑 → 安全检查 🟡
    /simplify 审查代码找简化机会 代码跑通了但觉得太复杂 → /simplify 找冗余 🟢
    /diff 交互式 diff 查看器 改了 10 个文件 → /diff 逐个看改了什么 🟢

    与 ae-skills 的区别：/code-review 审查的是未提交的 diff（轻量快速）。ae-skills 的 code-review-and-quality 审查的是任意代码（深度五维度）。两者互补——提交前用 /code-review，深度审查用 skill。

    四、状态与监控（6 个）¶

    了解当前会话的用量和上下文状态。

    命令 用途 AE 场景 频率
    /cost 查看 token 用量统计 "这次排查花了多少 token？" 🟡
    /context 上下文使用可视化（彩色网格） 长对话变慢了 → /context 看还剩多少空间 🟡
    /insights 生成本次会话的分析报告 回顾本次会话的效率、发现、改进点 🟢
    /debug 开启 debug 日志 AI 行为异常 → /debug 记录日志发给 Anthropic 🟢
    /heapdump 写堆快照诊断内存 Claude Code 本身内存占用过高 ⚪
    /fast 切换快速模式 简单回答不需要深度推理 → /fast on 更快 🟢

    五、工作流与自动化（7 个）¶

    多轮迭代、批量操作、定时任务。

    命令 用途 AE 场景 频率
    /plan 进入 Plan Mode 复杂改动先出方案——详见 2-9 Plan Mode 🔴
    /goal 设定目标，AI 跨多轮自动迭代直到满足 "让所有测试通过" → AI 修改→编译→测试→循环直到全绿。详见 2-9 🟡
    /loop 定时重复执行某个 prompt "每 10 分钟检查一次 CI 状态，失败了通知我" 🟡
    /batch 大规模编排跨代码库的改动 重构：改 30 个文件的 API 调用 → /batch 自动批量处理 🟡
    /fork 派生一个后台子 Agent "fork 一个 agent 去审查所有 ISR 函数的安全问题" 🟢
    /deep-research 多源搜索 → 交叉验证 → 输出带引用报告 选型对比："ESP32 vs STM32 低功耗方案" 🟡
    /schedule 创建/管理云端的定时任务 每天晚上自动跑一遍项目的完整测试套件 🟢

    六、IDE 与集成（7 个）¶

    Claude Code 和外部的连接。

    命令 用途 AE 场景 频率
    /ide 管理 IDE 集成（VS Code / JetBrains） 终端改好代码 → /ide 在 VS Code 里看 diff → 手动微调 🟡
    /plugin 管理插件（安装/启用/禁用/列表） "装 ae-skills 插件" → /plugin install ae-skills 🟡
    /mcp 管理 MCP server 连接和 OAuth codegraph 连不上 → /mcp 查看状态 → 重连 🟡
    /keybindings 打开键盘快捷键配置文件 绑一个快捷键给 /code-review 🟢
    /terminal-setup 配置终端快捷键（Shift+Enter 等） 终端里 Shift+Enter 不生效 → /terminal-setup 修复 🟢
    /install-github-app 安装 Claude GitHub App 让 Claude 自动审查 PR ⚪
    /install-slack-app 安装 Claude Slack App Slack 里召唤 Claude ⚪

    七、桌面与远程（4 个）¶

    跨设备、跨平台使用。

    命令 用途 AE 场景 频率
    /teleport 把当前会话发送到 claude.ai/code 网页端 终端排查太累 → 发到网页继续看 🟢
    /desktop 在 Claude Code Desktop 中继续当前会话 从终端切到桌面 App 🟢
    /remote-control 让 claude.ai 可以远程访问当前会话 不在电脑前时用手机继续 ⚪
    /add-dir 添加额外的工作目录 项目引用了另一个仓库的代码 → 加进工作区 🟢

    八、账号与反馈（5 个）¶

    命令 用途
    /login / /logout 登录/登出 Anthropic 账号
    /feedback 提交 bug report 或反馈
    /release-notes 查看版本更新日志
    /help 列出所有可用命令
    /exit 退出 Claude Code CLI

    九、其他（5 个）¶

    命令 用途 备注
    /agents 管理 Agent 配置 自定义 sub-agent 类型
    /hooks 查看 hook 配置 查看哪些生命周期事件有 hook
    /vim Vim 编辑模式 习惯 vim 的工程师
    /focus 简化显示模式 减少视觉噪音
    /color 设置提示栏颜色 个性化
    /copy 复制上一次 AI 回复到剪贴板 快速分享
    /btw 问一个不进入历史的附带问题 不影响对话主线的快速查询

    AE 工程师的 Top 10 必学命令¶

    按每天使用的频率排序：

    # 命令 一句话
    1 /clear 清空上下文，开始新任务
    2 /compact 长对话压缩——排查 1 小时后必用
    3 /plan 复杂改动先出方案
    4 /permissions 配置好了就不再弹确认框
    5 /cost "这次排查花了多少 token？"
    6 /review 审查同事的 PR
    7 /rewind AI 改错了 → 一键回退
    8 /doctor 什么东西不工作了 → 诊断
    9 /init 新项目第一件事
    10 /deep-research 选型对比、技术调研

    与 ae-skills 的配合¶

    很多斜杠命令和 ae-skills 有重叠——知道什么时候用哪个：

    场景 用斜杠命令 用 ae-skills
    审查未提交的改动 /code-review —
    审查任意代码（含已提交） — code-review-and-quality
    审查 PR /review —
    写规格 — spec-driven-development
    出方案 /plan planning-and-task-breakdown
    调试 — debugging-and-error-recovery
    查 datasheet — source-driven-development
    选型对比 /deep-research idea-refine

    原则：斜杠命令 = 通用能力（所有项目都能用）。ae-skills = 领域能力（嵌入式/AE 场景深度优化）。两者不冲突——斜杠命令快，skill 深。
